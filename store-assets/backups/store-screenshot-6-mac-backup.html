Quick reference guide for class structures and object relationships.

1. OOP Fundamentals

Class = blueprint for creating instances. Object = the actual instance.
  • • Inheritance: Child classes reuse methods and fields from a parent class.
  • • Polymorphism: Overriding parent methods to customize behavior in child classes.
  • • Encapsulation: Restricting direct access to some of an object's internal components.
  • • Abstraction: Hiding implementation details and showing only essential features.
