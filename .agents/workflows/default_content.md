# Workflow: Default Onboarding Content

## Trigger
User types: `/default_content`

## Steps

1. **Analyze**: Run `analyze_default_content` to examine initial local storage loading routines.
2. **Plan**: Run `plan_default_content` to map out the template variables and insertion logic.
3. **Execute**: Run `execute_default_content` to edit `notepad.tsx` and `App.tsx`.
4. **Verify**: Run `pnpm run build` from the workspace root to check for build errors.
5. **Finalize**: Record the history log under `_changelog/history/` and update `_tasks/QUEUE.md`.
